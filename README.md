## 📌 Overview
**Blockchain-Tender-DApp** is a decentralized Ethereum-based application (DApp) for managing tenders and contracts securely.  
It enables transparent bidding, voting, and automatic winner selection using smart contracts and MetaMask integration.  
Built for property and contract tenders, it ensures fairness, transparency, and decentralization.

---

## 🚀 Features
- Pre-registered users can create tenders  
- Open bidding for all Ethereum addresses  
- One bid per address with revision support  
- Voting system (upvote/downvote)  
- Automatic winner selection after bidding deadline  
- Admin-controlled user registration  
- MetaMask wallet integration  
- Responsive frontend interface with real-time updates  

---

## 🛠️ Technologies Used
- Solidity (Smart Contracts)  
- Ethereum Blockchain  
- Ganache (Local Blockchain)  
- MetaMask (Wallet Integration)  
- React.js (Frontend)  
- Web3.js / Ethers.js (Blockchain Interaction)  

---


## 👤 User Roles

### Admin
- Approve or reject new user registrations
- Manage system settings and registered users

### Registered Users
- Create new tenders
- View and vote on tenders
- Place and revise bids

### Non-Registered/Public Users
- Browse tenders
- Place bids (if allowed)
- Vote on tenders


## 📖 User Guide

**Quick Steps:**  
1. Connect MetaMask wallet  
2. Check your status (Registered / Not Registered)  
3. Browse tenders and place or increase bids  
4. Vote on tenders you didn’t create  
5. Registered users can create new tenders  
6. Admin approves new user registration requests  
7. Winner automatically assigned after bidding ends

## 🔐 Key Functional Logic
- One bid per address enforced  
- Bid revisions allowed until deadline  
- Smart contract automatically selects the highest bidder  
- Voting system ensures transparency and fairness  
- All transactions recorded on blockchain for full auditability
